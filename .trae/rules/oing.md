# Gomini 项目统一规范（唯一规则，无冲突）
## 1. 核心总原则
**允许：新增、追加、扩展、优化、拆分、在指定模块内修改。
禁止：删除、重构、覆盖、重写、破坏原有功能。**

## 2. 绝对禁止修改（任何文件都不能动）
- 禁止修改 index.html 已有 DOM、id、class
- 禁止修改 CONFIG、常量、映射表、基础配置
- 禁止修改 State 原有结构
- 禁止修改 Storage 存储 KEY、数据结构
- 禁止覆盖/删除原有函数
- 禁止破坏样式基础（CSS变量、安全区、导航）

## 3. 允许新增（全部允许）
- 允许新增视图 /views
- 允许新增模块 /modules
- 允许新增函数、样式、逻辑
- 允许追加代码、注释
- 允许优化代码风格、精简冗余

## 4. 允许【指定文件内修改】（唯一例外，不冲突）
**以下文件允许内部追加/修改，用于统一维护功能：**
- /modules/event.js 允许添加/修改事件处理
- /modules/business.js 允许追加业务逻辑
- /modules/render.js 允许追加渲染方法
- style.css 允许新增/调整样式
- app.js 允许注册路由、配置视图

## 5. 拆分规则
- 允许按规范拆分 JS 到 /modules 和 /views
- 只移动代码，不修改逻辑、不删除功能
