## 一、数据输入格式

假设历史开奖数据格式如下（按期次降序排列，索引 0 为最新）：

```javascript
const historyData = [
  { expect: 2026134, openCode: "01,05,12,23,34,45,07", zodiac: "鼠,虎,马,羊,猴,鸡,狗" },
  { expect: 2026133, openCode: "...", zodiac: "..." },
  // ...
];
````

- `expect`：期数（数值，用于判断新旧顺序）
- `zodiac`：字符串，逗号分隔 7 个生肖（第 7 个是特码生肖）

## 二、组件功能需求

### 1. 多窗口统计

- 支持三个窗口大小：**12 期、24 期、36 期**。
- 每个窗口独立统计最近 `W` 期的数据（取 `historyData.slice(0, W)`）。

### 2. 对每个生肖（12 个生肖）计算以下指标

- `count`：在窗口内作为特码出现的次数。
- `miss`：当前遗漏期数（距离上一次出现的间隔，若从未出现则等于窗口大小）。
- `zone`：根据 `count` 划分区域：

  text
  ```
  count === 0 → '等待区'
  count === 1 → '低频区'
  count === 2 → '中频区'
  count === 3 → '高频区'
  count >= 4 → '顶峰区'
  ```
- `willDrop`：布尔值，表示下一次窗口滑动时（新数据加入，最旧数据移出）该生肖的次数是否会减少。
  - 判断条件：在窗口内，该生肖**最早出现的期次** === 窗口内**最旧的数据期次**（即 `windowData[W-1].expect`）。
  - 如果生肖在窗口内未出现，则 `willDrop = false`。

### 3. 状态存储（用于显示区域变化）

- 对每个窗口（12/24/36），需要缓存上一次（前一次数据刷新时）每个生肖所在的区域 `prevZone`。
- 存储方式：`localStorage`，key 分别为 `ZONE_PREV_ZONE_p12`、`ZONE_PREV_ZONE_p24`、`ZONE_PREV_ZONE_p36`。
- 初次运行时，`prevZone` 等于当前区域。

### 4. 实时刷新逻辑

- 当外部更新 `historyData` 后，重新计算所有窗口数据，并更新 UI。
- 同时更新 localStorage 中的 `prevZone`（如果当前区域与缓存不同，则覆盖）。

## 三、UI 布局与结构（完全复刻）

### 整体布局

- 三个独立卡片（使用 `.freq-card` 类），分别对应 12 期、24 期、36 期。
- 每个卡片内部**按区域分组显示**，区域顺序固定为：

  text
  ```
  顶峰区 → 高频区 → 中频区 → 低频区 → 等待区
  ```
- 每个区域下面展示属于该区域的生肖卡片网格（网格自适应，最小宽度约 80px）。

### 单个生肖卡片（`.zone-zod-card`）包含以下元素：

1. **右上角次数徽章**（`.zod-card-count-badge`）：显示 `count`。
   - 如果 `willDrop === true`，在次数旁边显示一个红色向下箭头 🔻 或 SVG 箭头。
2. **生肖名称**（`.zod-card-name`）：
   - 左侧显示一个小圆点（`.prev-zone-dot`），圆点颜色 = 该生肖**上一次区域**对应的颜色（从 localStorage 读取）。
   - 圆点右边是生肖文字（如“马”）。
3. **遗漏期数**（`.zod-card-miss`）：
   - 显示“X期”。
   - 背景色 = 上一次区域对应的颜色（浅色半透明），文字颜色加深。

### 区域标签样式

- 每个区域标题左侧有一个彩色标签（`.freq-zone-tag`），背景色为该区域主题色，文字白色。
- 右侧显示该区域生肖数量（`.zone-count-badge`）。
