# 项目强制开发规则
版本：2.0
适用对象：所有开发、修改、重构
强制执行：所有代码必须遵守
# 1. 层级架构（绝对不能乱）
采用 单向分层 架构：
  视图层 View → 业务层 Business → 工具层/状态层/存储层
规则：
1. View 可以调用 Business
2. Business 严禁调用 View
3. Business 严禁操作 DOM
4. 视图只管展示，业务只管计算
5. 禁止跨层、反向、循环依赖
# 2. 文件职责（严禁越界）
## business.js（纯业务/数据层）
允许：
- 数据统计、概率计算、算法逻辑
- 冷热、遗漏、五行、生肖、特码分析
- 数据格式化、清洗、转换
- 存储层封装调用
- 纯函数、无副作用
严禁：
- document.getElementById、querySelector
- innerHTML、appendChild、DOM 操作
- classList、show、hide、style、样式修改
- HTML 渲染、拼接、模板生成
- 事件绑定、点击、交互逻辑
- Toast、弹窗、提示、UI 反馈
## view-analysis.js（分析页面专属）
允许：
- 分析页 DOM 渲染
- 卡片排序、列表展示
- 分析页交互、事件
- 调用 business 获取数据
- 分析页专属 UI 逻辑
## view-filter.js（筛选页面专属）
允许：
- 筛选页渲染
- 筛选标签、选项交互
- 筛选结果展示
- 筛选页逻辑
## view-lottery.js（记录/彩票页面专属）
允许：
- 记录列表渲染
- 历史、精选、预测数据展示
- 卡片样式、排序、筛选
- 记录页交互、删除、复制
## app.js（入口调度）
允许：
- 应用初始化
- 模块调度
- 页面启动流程
禁止：
- 业务逻辑
- 渲染、DOM、事件

## style.css
允许：
- 全局样式、主题、UI 外观
禁止：
- 逻辑、行为、业务

## dom.js / event.js / state.js / storage.js / utils.js / toast.js
允许：
- 单一职责工具功能
禁止：
- 业务逻辑、渲染逻辑、页面逻辑

## script.js
状态：已废弃，严禁引用、严禁修改、严禁使用
# 3. 代码禁区（绝对禁止）
1. 禁止在 business.js 操作任何 DOM
2. 禁止在 business.js 写任何渲染
3. 禁止把视图逻辑写进 business
4. 禁止把业务算法写进 view
5. 禁止创建未在规则中定义的文件
6. 禁止跨页面逻辑污染
7. 禁止在视图层写复杂统计算法
# 4. 新增功能流程（必须遵守）
1. 判定是展示 → 写 view
2. 判定是计算 → 写 business
3. 视图只调用业务，业务不依赖视图
4. 不越层、不越权、不乱加
# 5. 每次修改必须自检（强制）
1. 我是否把视图逻辑写到 business 了？
2. 我是否在 business 里操作了 DOM？
3. 渲染/排序/事件是否都在 view？
4. 是否没有越界、没有违规？
# 6. AI 开发者承诺（必须遵守）
我将严格遵守以上全部规则，
绝不越界、绝不乱加、绝不污染架构。
